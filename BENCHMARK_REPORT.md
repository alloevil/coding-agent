## Bug Found & Fixed

### Issue: `tdd_ops.py` hardcodes `python` instead of `sys.executable`

**Symptom**: On systems where only `python3` exists (Debian/Ubuntu), `python -m pytest` fails with "command not found". The TDD fix loop silently exits with `passed=0, failed=0`, thinking all tests passed.

**Impact**: 
- 4 test cases in `test_tdd_fix_loop.py` gave false-positive results
- `tdd_fix_loop` tool was completely broken on `python3`-only systems

**Fix** (PR #27):
```python
import sys
_PY = sys.executable or "python3"

FRAMEWORK_COMMANDS = {
    "pytest": f"{_PY} -m pytest {{path}} -v --tb=short 2>&1",
    ...
}
```
